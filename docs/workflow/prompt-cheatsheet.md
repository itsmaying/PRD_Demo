# 产品工作流常用指令速查

## 开始讨论一个新需求

```text
我有一个新需求 idea：……
先不要实现，我们先讨论清楚。
```

```text
我们先进入需求讨论阶段，不改代码。
我想做的需求是：……
请先帮我澄清场景、用户目标、页面范围和关键交互。
```

## 切换模型或新会话后读取上下文

```text
请先读取：
- docs/workflow/00-overview.md
- 当前需求目录下的 README.md、prd.md、decision-log.md、version-log.md

读取后先总结你理解到的上下文，再继续。
```

如果还没有需求目录：

```text
这是一个新需求，目前还没有目录。
请先读取 docs/workflow/00-overview.md，然后只和我讨论需求，不要创建文件或改代码。
```

## 讨论中途总结

```text
先不要实现。
请把目前讨论出的结论、未确定问题、推荐方向整理一下。
```

```text
请按以下结构总结当前需求讨论：
1. 已确认结论
2. 待确认问题
3. 推荐方案
4. 不做的范围
5. 后续如果实现，需要生成哪些文件
```

## 确认可以实现

```text
方向确认，开始实现。
请按照刚才讨论的结论创建目录和文件。
```

```text
可以实现了。
请生成这个需求的文件夹、PRD、decision-log、version-log，并接入 Demo Hub。
```

## 只生成 PRD

```text
这个需求先不做 demo。
请只基于讨论结论生成评审用 PRD。
```

## 只做 Demo

```text
这个阶段先做交互 demo。
PRD 先不要完整生成，只记录关键结论和待确认问题。
```

## 更新已有 Demo

```text
我要迭代已有 demo：xxx。
请先读取该 demo 的 README.md、prd.md、decision-log.md、version-log.md。
读取后总结当前版本，再和我讨论这次要改什么，不要直接实现。
```

## 记录关键决策

```text
请把刚才这个结论记录到当前 demo 的 decision-log.md。
```

```text
这个决策后续很重要，请留档：……
```

## 准备需求评审

```text
这个 demo 已经满意了。
请基于当前 demo 和 decision-log，整理一版适合需求评审会使用的 PRD。
```

## 生成同事查看说明

```text
请生成/更新这个 demo 的查看说明。
说明同事如何访问 Demo Hub、手机如何添加到桌面、源码和 PRD 在哪里。
```
