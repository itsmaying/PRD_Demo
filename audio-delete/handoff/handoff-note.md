# 录音删除管理 Handoff

## 交付方式

对开发侧的实际交付文件只有一个：

- `AudioDeleteHandoff.jsx`

不额外要求他们阅读前端工程内的其他源码文件。

## 文件定位

`AudioDeleteHandoff.jsx` 是给安卓 / iOS 开发看的单文件交互参考实现。
目标是让开发只看这一个文件，就能理解当前 Demo 的页面结构、状态切换、交互路径和关键文案。

## 一致性说明

`AudioDeleteHandoff.jsx` 以当前 React Demo 为准做 1:1 行为映射，保留了：

- 默认进入未关联 Tab。
- 50 条 mock 数据、分页加载和“继续下滑加载更多”提示。
- 列表页搜索栏、顶部三个图标、Tab、右侧筛选箭头。
- 左滑进入批量管理、进入后自动选中当前录音。
- 已关联录音在当前 Demo 中的提示文案：`录音已关联客户，不可操作`。
- 编辑模式顶部栏、全选逻辑、底部重命名 / 删除操作栏。
- 删除确认弹窗文案：`删除后无法恢复，是否继续？`。
- 重命名弹窗标题：`命名当前文件`。
- 详情页客户 pill、播放器区、三个内容 Tab、右上角更多按钮和底部删除 sheet。
- 删除成功 / 重命名成功 Toast。

## 使用提醒

这个单文件的目的是减少阅读工程拆分成本，不是重新定义交互。
