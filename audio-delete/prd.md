# 录音删除管理 PRD

## 1. 需求背景

用户在日常使用录音功能后，会产生一些无效、重复或不再需要保留在当前工作视图中的录音。如果录音长期累积，会影响用户查找有效录音，也增加列表管理成本。

本需求中的“删除”，本质上优先解决一线销售的录音管理诉求：把无效、重复、暂不需要的录音从个人列表中清理出去，让当前工作视图更清晰。

但录音可能已经关联到客户或业务记录，也可能对企业侧的客户跟进复盘、质检和业务风险追踪有价值。因此，本期删除能力需要优先保证安全边界：

```text
只允许删除未关联录音；已关联录音不可删除。
```

本需求通过列表页批量管理和详情页删除入口，支持用户清理未关联录音，同时避免误删已关联录音。

## 2. 需求目标

### 2.1 用户目标

- 快速清理不需要的未关联录音。
- 在录音较多时，可以批量选择和批量删除。
- 明确知道哪些录音不能删除，以及不能删除的原因。
- 删除前有二次确认，降低误操作风险。

### 2.2 产品目标

- 建立录音清理和管理能力，而不是单纯的数据销毁能力。
- 明确“已关联录音不可删除”的业务边界。
- 统一列表页和详情页的删除确认逻辑。
- 采用软删除策略：销售端删除后不可见，后台保留删除状态，管理端可按权限追踪。
- 提供可在手机端演示的交互 Demo，用于需求评审和方案确认。

## 3. 用户范围

目标用户为使用录音功能的一线业务人员，例如销售、客户经理、客户跟进人员等。

典型场景：

- 用户录制后发现录音无效，希望删除。
- 用户整理历史录音，希望批量清理未关联录音。
- 用户查看某条录音详情后，确认该录音不需要保留，希望从详情页删除。
- 用户尝试删除已关联客户的录音时，需要被明确阻止。

## 4. 页面范围

本期覆盖两个核心页面：

### 4.1 录音列表页

页面包含：

- 顶部搜索区域；
- Tab 筛选：全部 / 已关联 / 未关联；
- 录音卡片列表；
- 左滑进入批量管理；
- 编辑模式顶部栏；
- 底部批量操作栏；
- 删除确认弹窗；
- 重命名弹窗；
- Toast 反馈。

### 4.2 录音详情页

页面包含：

- 返回按钮；
- 客户关联状态入口；
- 音频播放区域；
- 录音标题和基础信息；
- 内容 Tab；
- 右上角更多操作；
- 底部操作面板；
- 删除确认弹窗；
- Toast 反馈。

## 5. 核心业务规则

## 5.1 删除权限规则

| 录音状态 | 是否允许删除 | 页面表现 |
| --- | --- | --- |
| 未关联录音 | 允许删除 | 可左滑、可选中、可批量删除、详情页可删除 |
| 已关联录音 | 不允许删除 | 不提供删除入口，不参与全选，点击时提示不可删除 |

规则说明：

- 只有“未关联”录音允许删除。
- “已关联”录音不允许删除。
- 全选只选择未关联录音。
- 编辑模式中点击已关联录音时，提示：`录音已关联客户，不可删除`。

## 5.2 删除策略规则

本期采用软删除策略，不做自动硬删。

| 视角 | 删除后表现 |
| --- | --- |
| 一线销售端 | 从录音列表、搜索结果和当前工作视图中移除，用户感知为已删除 |
| 系统后台 | 不物理删除录音文件和记录，仅标记删除状态 |
| 管理端 | 可按权限查看或分析已删除录音，并展示“销售已删除”状态 |

规则说明：

- 删除的核心目标是帮助一线销售清理个人工作视图，而不是立即销毁企业侧业务数据。
- 系统需记录删除人、删除时间、删除来源和删除状态。
- 本期不设置 30 天自动硬删等固定清理周期。
- 如后续存在明确合规、客户配置或存储治理要求，再补充永久删除、保留周期或企业级删除策略。

## 5.3 Tab 范围规则

### 全部

- 展示全部录音。
- 未关联录音支持左滑进入编辑模式。
- 已关联录音不支持删除。
- 编辑模式中只允许选择未关联录音。

### 已关联

- 展示已关联录音。
- 不支持左滑。
- 不支持删除。
- 不展示删除入口。

### 未关联

- 展示未关联录音。
- 支持左滑进入编辑模式。
- 支持选择、全选、批量删除。
- 支持单条重命名。

## 6. 列表页交互规则

## 6.1 左滑进入批量管理

用户在未关联录音卡片上左滑时：

```text
不弹出单条操作菜单，直接进入编辑 / 批量管理模式。
```

进入编辑模式后：

- 被左滑的录音自动选中；
- 页面顶部切换为编辑模式顶部栏；
- 页面底部展示批量操作栏；
- 每条可选录音右侧展示选择状态。

采用该方案的原因：

- 减少操作路径；
- 更适合批量清理录音；
- 避免“左滑菜单”和“批量管理”两套入口并存导致体验复杂。

## 6.2 编辑模式顶部栏

顶部栏规则：

| 区域 | 内容 | 交互 |
| --- | --- | --- |
| 左侧 | `X` | 退出编辑模式 |
| 中间 | `已选中 N 个` | 展示当前选中数量 |
| 右侧 | `全选` | 选择全部未关联录音 |

退出编辑模式时：

- 清空已选中录音；
- 隐藏底部批量操作栏；
- 回到普通列表状态。

## 6.3 全选规则

点击全选时：

- 只选择当前列表范围内的未关联录音；
- 不选择已关联录音；
- 如果当前已经选中全部可选录音，再次点击则取消全选。

## 6.4 底部批量操作栏

底部操作包含：

| 操作 | 可用条件 | 说明 |
| --- | --- | --- |
| 重命名 | 仅选中 1 条录音 | 打开重命名弹窗 |
| 删除 | 选中数量大于 0 | 打开删除确认弹窗 |

不可用状态：

- 未选中录音时，删除不可用；
- 选中多条录音时，重命名不可用。

## 7. 删除流程

## 7.1 删除触发入口

删除入口包括：

- 列表页底部批量操作栏中的 `删除`；
- 详情页右上角 `...` 中的 `删除录音`。

## 7.2 删除二次确认

所有删除动作都必须二次确认。

弹窗规则：

- 使用自定义弹窗，不使用系统原生 confirm；
- 标题和正文左边界对齐；
- 标题明确提示删除；
- 正文提示删除后将从当前列表中移除；
- 按钮为：取消 / 确定。

弹窗文案：

```text
标题：确认删除录音？
正文：删除后将从当前列表中移除，是否继续？
```

## 7.3 删除成功反馈

用户确认删除后：

- 从销售端当前列表中移除对应录音；
- 退出编辑模式；
- 清空选中状态；
- 如果从详情页删除，则返回列表页；
- 后台保留录音数据，并记录删除状态；
- 显示 Toast：`删除成功`。

## 7.4 已关联录音不可删除反馈

当用户在编辑模式中点击已关联录音时，系统不允许选中，并提示：

```text
录音已关联客户，不可删除
```

## 8. 重命名流程

## 8.1 入口

重命名入口位于列表页编辑模式底部操作栏。

## 8.2 可用条件

- 仅选中 1 条录音时可用；
- 未选中或选中多条时不可用。

## 8.3 交互规则

点击重命名后：

- 打开自定义重命名弹窗；
- 输入框默认带入当前录音名称；
- 用户确认后更新录音名称；
- 显示 Toast：`重命名成功`。

## 9. 详情页交互规则

## 9.1 客户关联状态

详情页顶部展示当前录音的客户关联状态：

| 状态 | 展示 |
| --- | --- |
| 未关联 | `未关联客户` |
| 已关联 | 客户名称 |

当前 Demo 中，该入口仅作为占位交互，不展开客户绑定完整流程。

## 9.2 更多操作入口

点击右上角 `...` 后，呼出底部操作面板。

底部操作面板规则：

- 使用 App 化底部圆角卡片；
- 不展示灰色拖拽条；
- 不展示显式取消按钮；
- 点击遮罩空白区域关闭；
- 删除项为白底通栏可点击区域；
- 图标 + `删除录音` 文案整体居中；
- 仅使用红色图标和红色文字表达危险操作。

## 9.3 详情页删除

- 仅未关联录音展示删除入口；
- 点击 `删除录音` 后，关闭底部操作面板，并打开统一删除确认弹窗；
- 确认删除后，返回录音列表页；
- 显示 Toast：`删除成功`。

## 10. 视觉与体验要求

整体体验应更接近移动端 App，而不是 Web 管理后台。

重点要求：

- 卡片式录音列表；
- 圆角和轻阴影；
- 紫色作为主色；
- 危险操作使用红色表达；
- 删除确认使用自定义弹窗；
- 底部操作使用 App 化浮层；
- Toast 用于轻量反馈；
- 删除等高风险操作必须有二次确认。

## 12. 本期不做范围

本期不处理：

- 销售端最近删除 / 回收站；
- 销售端删除撤销；
- 自动硬删或固定清理周期；
- 权限系统；
- 接口设计；
- 真实音频播放能力；
- 客户绑定完整流程；
- Android / iOS 翻译实现建议；
- 工程侧具体实现方案。

## 13. 待确认问题

以下问题不影响当前 Demo 版本评审，可在后续版本继续讨论：

- 管理端如何展示“销售已删除”状态；
- 管理端是否支持筛选已删除录音；
- 是否需要销售端最近删除 / 回收站；
- 是否允许销售端恢复已删除录音；
- 是否需要企业级永久删除或数据保留周期配置；
- 批量删除确认弹窗中是否需要展示具体删除数量；
- 重命名是否允许空名称；
- 重命名是否需要字数限制；
- 搜索结果中是否也支持左滑 / 编辑；
- Demo 是否需要保留多个版本入口；
- 长期部署选择 Vercel 还是 Cloudflare Pages。
