背景及目标
为 FindAI Air 耳机建立设备管理体系，确保：
1. 防伪：阻止非授权/克隆设备接入 APP。
2. 溯源：通过 SN 管理生产批次与
业务背景
为实现 FindAI Air 智能耳机的设备溯源、正品校验及售后管理，需要在生产阶段为每台硬件设备烧录唯一的“设备指纹”
核心目标
唯一识别： 确保每台设备拥有全球唯一的 16 位 SN 序列号。
鉴权防伪： 通过 24 位 Auth Code（认证码）实现设备与 APP 连接时的合法性校验。

目标范围
设备：FindAI Air 耳机
平台：iOS / Android APP

技术定义与规则
售后盒标 SN
- SN 主要是给售后和仓库看的，原则：一眼能看懂批次，全球唯一。
- 长度： 16 位字符串。
- 构成逻辑： 项目代号(3位) + 年份(2位) + 月份(1位) + 批次代码(2位) + 流水号(8位)。
  - FAA (FindAI Air)   
- 量产号段范围： FAA2640100000001 到 FAA2640100003000（共 3000 组）。
- 测试号段范围： FAR2639900000001 到 FAR2639900000005（SRRC 测试机专用）。

后台认证码 Auth Code
- 长度： 24 位。
- 行业惯例推荐： 大写字母+数字（16进制 A-F, 0-9）
- 生成规则： 通过SN+私钥 + HMAC-SHA256 算法生成
  -  Auth Code = HMAC-SHA256 (SN + Secret_Key) ，确保每个 SN 都有唯一对应的 Auth Code

业务流程
阶段一：数据生产流程
产出sn+auth Code
暂时无法在飞书文档外展示此内容
阶段二：APP校验流程
1. 读取信息： APP 通过蓝牙连接 FindAI Air 后，通过蓝牙协议读取固件里的 SN 和 Auth Code。
2. 云端验真：  APP 将接收到的信息传给后端比对
3. 校验逻辑：
  - 后端比对： 否存在这对 SN-AuthCode 组合
    - 验证 A： 该 SN 是否在 FindAI 的合法号段库内？
    - 验证 B： 该 Auth Code 是否与该 SN 匹配？（防止恶意克隆固件）
  - 状态反馈： 如果匹配，返回“正品已激活”；如果不匹配，APP 提示“非授权设备”。
  - 数据落库： 校验通过后端记录当前机器是哪个用户、什么时间第一次连接的（首次激活）
4. 触发时机：
  1. 仅在以下情况触发云端校验，日常连接不重复校验：
    1. 首次连接：设备从未与 APP 配对过。
    2. 重置后连接：设备恢复出厂设置后重新连接。
    3. 手动触发：用户在设备页面点击“重新验证”。
